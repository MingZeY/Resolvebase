# Resolvebase

> 一种基于图论的结构化问题求解方法论 —— 让 AI Agent 像遍历图一样思考。

Resolvebase 是一个面向 AI 智能体的元技能（Meta-Skill），它将问题求解抽象为**有向图上的状态转移**过程。通过精确定义状态、路径、权重与观测机制，使 AI Agent 能够以结构化、可验证、可回溯的方式解决任意复杂问题。

---

## 核心理念

Resolvebase 将一切问题建模为图论中的路径搜索：

```
[当前状态] ──动作──> [中间状态1] ──动作──> ... ──动作──> [目标状态]
```

| 概念 | 图论类比 | 说明 |
|------|----------|------|
| **状态 (State)** | 节点 (Node) | 可验证、可测量的具体情境 |
| **问题 (Problem)** | 路径未知 | 两个状态之间的连接模糊或缺失 |
| **路径/动作 (Method)** | 边 (Edge) | 促使状态转化的可执行操作 |
| **权重与约束** | 边权重 | 时间、算力、认知负荷等代价 |
| **观测与断言** | 节点验证 | 验证是否已到达预期状态 |
| **死路与回溯** | 剪枝/DFS回溯 | 失败作为合法的状态信息收集 |

### 不再模糊的问题定义

**糟糕** ❌：  
"把代码变得更好"、"学习一下这个框架"

**优秀** ✅：  
> 当前状态：代码在并发 1000 时抛出 Timeout 异常  
> 目标状态：代码在并发 1000 时响应时间 < 200ms  
> 动作：提取 50 行重复逻辑并封装为公共函数

---

## 特性

- **图论驱动的决策框架** —— 将模糊需求转化为可执行的状态转移路径
- **可验证的渐进式求解** —— 每个中间状态都必须可观测、可验证
- **支持并行与回溯** —— 并行探索多条路径，失败自动回溯到有效分叉点
- **跨上下文持久化** —— 通过 `.resolvebase` 状态空间在对话上下文之间保留知识
- **任务复杂度自动分级** —— 简单 / 中等 / 复杂任务分别采用不同的分解策略
- **冲突检测与澄清** —— 遇到模糊或矛盾的指令时主动寻求用户确认
- **平台无关** —— 纯文本 Skill 定义，可部署到任意支持 Skills 的智能体平台

## 部署

> **警告**：仅导入 Skill 定义文件是不够的。必须同时配置「强制引导规则」，否则 Agent 不会主动加载 Resolvebase。

### 结构概览

```
.
├── rules.md                  # 强制引导规则（关键文件，位置取决于平台）
├── resolvebase.md                    # Skill 主定义文件
├── .resolvebase/                   # 状态空间（元数据存储）
│   ├── define/                     # 定义文件：不可变的共享知识单元
│   ├── method/                     # 方法文件：可复用的操作流程
│   ├── state/                      # 状态文件：运行时快照
│   └── index/                      # 索引文件：层级目录系统
└── README.md
```

### 第一步：强制引导规则（必须）

Resolvebase 是一个元技能，Agent 本身不会主动使用它。你必须在平台规则中配置**强制引导**，要求 Agent 在每次对话开始时首先加载 Resolvebase，然后再处理用户请求。

以下为通用强制引导规则模板（请根据平台语法适配）：

```markdown
# Resolvebase
Resolvebase 是一种解决问题的基本思路，适用于回答问题、编程、调试等任何场景的 skill，可以帮助你更好的解决问题。
- 无论什么情况请立即调用工具读取 skill:resolvebase，再响应用户的问题
- 使用项目根目录下的 `.resolvebase` 文件夹作为你的状态空间
```

**核心要点**：规则必须包含两条指令：

| 指令 | 作用 |
|------|------|
| **先加载 Skill，再响应用户** | 确保 Agent 在开始任何工作前，已将 Resolvebase 方法论加载到当前上下文 |
| **使用 `.resolvebase/` 作为状态空间** | 确保 Agent 的状态持久化与索引机制正确工作 |

配置方式（以主流平台为例）：

| 平台 | 配置位置 |
|------|----------|
| **Hermes** | 在 `rules/` 目录下创建 `basic.md`，设置 `alwaysApply: true` |
| **Lobster** | 在 `rules/` 目录下创建 `basic.md`，设置 `alwaysApply: true` |
| **Cursor** | 在 `.cursor/rules/` 下创建规则文件，设置 `alwaysApply: true` |
| **Claude Code** | 在 `CLAUDE.md` 或 `.claude/settings.json` 中添加对应 rules 配置 |
| **其他 Agent 平台** | 找到平台的「始终生效规则 / System Prompt 前缀」机制，注入上述规则内容 |

> 仓库中 `rules/basic.md` 是强制引导规则的示例实现，可直接参考。

### 第二步：部署 Skill 定义

强制引导规则配置完成后，将 Resolvebase 的 Skill 定义部署到目标平台：

**Hermes / Lobster**  
将 `skills/resolvebase/SKILL.md` 放置到平台的 skills 目录下，配合上一步的强制引导规则即可生效。

**Cursor / Claude Code / 其他 Agent 平台**  
拷贝 `SKILL.md` 的核心内容（`## 定义` 至 `## 注意事项`），粘贴到目标平台的 Skill 配置中。`.resolvebase/` 目录作为状态空间需置于项目根目录下。

**自定义 Agent / LangChain / AutoGPT**  
将 Resolvebase 的工作流伪代码（`SKILL.md` 中的 `## 实践 > ### 工作流` 部分）翻译为目标 Agent 框架的执行逻辑，`.resolvebase/` 作为持久化存储层。

### 快速开始

```bash
# 1. 克隆仓库
git clone https://github.com/MingZeY/Resolvebase.git
cd Resolvebase

# 2. 配置强制引导规则（关键步骤！）
# 根据上方「强制引导规则」章节，在你的 Agent 平台中创建对应的规则文件

# 3. 导入 Skill 定义
# 根据上方「部署到不同智能体平台」章节，将 SKILL.md 导入你的 Agent

# 4. 验证
# 向 Agent 发送任意问题，观察其是否首先加载 Resolvebase，然后按方法论执行
```

## 工作流

Resolvebase 的核心工作流遵循以下伪代码逻辑(详细逻辑参考resolvebase.md)：

```
确定当前状态 → 确定目标状态 → 收集数据与定义
    → 探索路径（前向/反向推导）
    → 并行执行 | 逐步迭代
    → 每步验证是否更接近目标
    → 失败则回溯，成功则持久化状态
```

关键约束：
- **任务复杂度强制分级**：简单任务可直接完成；中等/复杂任务必须分解为中间状态
- **重构类或多文件任务必须视为复杂任务**进行分解
- **每个中间状态必须保证可运行性和可验证性**（最佳中间态原则）

## 状态空间

`.resolvebase/` 是 Resolvebase 的持久化层，用于跨越对话上下文保留关键信息：

| 目录 | 用途 | 示例 |
|------|------|------|
| `define/` | 不可变的共享知识（项目规范、架构约定） | 编码规范、命名规则 |
| `method/` | 可复用的操作流程（输入→过程→输出） | 执行协议、文件管理流程 |
| `state/` | 运行时快照（任务进度、重构状态） | 当前重构进度、已完成的步骤 |
| `index/` | 层级索引系统（快速定位文件） | 定义索引、方法索引 |

> **注意**：状态空间不应存储公开可查的信息（如 API 文档）、完整源代码镜像等。
