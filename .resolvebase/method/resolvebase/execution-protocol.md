# 执行协议

## 输入
- `方法名称`：当前正在执行的 [[define/resolvebase/method|方法]]名称
- `方法步骤`：方法的 `## 过程` 章节中定义的所有步骤列表

## 输出
- 用户被明确告知正在执行的方法名称
- 每个步骤通过 TodoWrite 工具跟踪并实时更新执行状态；若 TodoWrite 不可用，降级为纯文本 TODO 格式（见步骤 7）
- 或：步骤执行失败 + 失败原因及已完成步骤列表

## 过程

1. **宣告方法**：在执行方法的第一步，向用户明确输出 "正在执行 method: <方法名称>"。不得跳过此步骤开始任何实际操作。

2. **解析步骤**：读取方法的 `## 过程` 章节，提取每个编号步骤（如 `1. `、`2. `）的内容描述，作为独立的 todo 条目。

3. **初始化 Todo 列表**：
   - 优先使用 TodoWrite 工具创建 todo 列表（merge=false），将解析出的每个步骤作为一个 todo 条目，按步骤顺序排列，所有条目初始状态为 `pending`
   - 若 TodoWrite 工具不可用，则使用纯文本 TODO 格式（格式规范见步骤 7）输出 todo 列表

4. **按序执行步骤**：从第一个步骤开始，依次执行：
   - 将当前步骤状态更新为 `in_progress`（优先 TodoWrite merge=true，无则输出纯文本 TODO 状态行）
   - 执行该步骤所描述的实际操作
   - **多级方法调用（子方法展开）**：若当前步骤在执行过程中触发了对其他 [[define/resolvebase/method|方法]]的调用，则读取被调用方法的 `## 过程` 章节，将其步骤展开并扁平插入到当前 TODO 列表（紧随当前步骤之后）。子方法步骤自身可能进一步触发方法调用，递归展开，全部纳入同一个扁平 TODO 列表
   - 执行完毕后，将当前步骤状态更新为 `completed`（优先 TodoWrite merge=true + summary，无则输出纯文本 TODO 状态行），在 summary 中简述执行结果

5. **步骤失败处理**：若任一步骤执行失败，将该步骤标记为 `completed`（summary 中记录失败原因），终止后续步骤执行，返回失败。

6. **完成确认**：全部步骤执行完毕后，确认所有条目状态均为 `completed`。

7. **纯文本 TODO 格式**：当 TodoWrite 工具不可用时，使用以下纯文本格式替代：
   ```
   - [ ] 步骤描述    ← pending（待执行）
   - [/] 步骤描述    ← in_progress（执行中）
   - [x] 步骤描述    ← completed（已完成，可附加 : 原因/结果）
   ```
   格式规则：
   - 每行一个条目，以 `- [状态符] ` 开头
   - 状态符三种：` `（空格）= pending，`/` = in_progress，`x` = completed
   - completed 条目可在描述后追加 `: 简述` 作为 summary 等价物
   - 子方法展开的步骤与父方法步骤使用同一缩进层级，扁平排列
