# 校验状态文件语法

## 输入
- `目标`：待校验的 [[define/resolvebase/state|状态]]文件路径

## 输出
- 校验通过
- 或：校验失败 + 违规项列表

## 过程

1. 读取 `目标` 文件，按行解析。
2. 检查首行是否以 `# ` 开头。否则记录 `缺少名称行`。
3. 检查是否无 `## 依赖` 章节。若有 → 记录 `禁止独立依赖章节，依赖须以 [[link]] 内联在正文中`。
4. 检查是否按顺序包含以下章节（`## ` 开头）：
   - `## 类型`（可选，含时须有 `[[define/...]]` 内联引用）
   - `## 快照`（必选，含"更新时间"和"当前状态"）
   - `## 目标`（必选）
   - `## 上下文`（必选，约束见步骤 11-12）
   - `## 关联`（可选，含时 `[[link]]` 仅限 `define/`/`method/`/`state/`）
   - `## 日志`（必选，条目须 `- [<ISO日期>] <描述>` 格式）
5. 缺少必选章节 → 记录 `缺少章节：<章节名>`。
6. 有上述列表之外的 `## ` 章节（除名称行和上述 6 种外） → 记录 `多余章节：<章节名>`。
7. 检查 `## 快照 > 当前状态` 是否可验证（能客观判断真假），非模糊感受。若模糊 → 记录 `当前状态不可验证`。
8. 检查 `## 日志` 条目是否带 ISO 日期。缺失日期的条目记录。
9. 链接语法校验（遵循 [[define/resolvebase/link-syntax|链接语法]]）：提取全文所有 `[[...]]` 模式，检查：
   - 若出现 `[[...]](...)` → 记录 `链接语法错误：应使用管道符 [[path|text]]`
   - 若出现 `[...](...)` (Markdown 链接) → 记录 `链接语法错误：应使用 [[wikilink]]`
10. 文件命名校验（遵循 [[define/resolvebase/file-naming|文件命名规则]]）：
    - 若 `目标` 属于 Resolvebase 系统文件路径，文件名须为小写英文连字符格式。否则 → 记录 `命名违规：系统文件须英文命名`
    - 若 `目标` 不属于系统文件路径，文件名须符合用户语言习惯。否则 → 记录 `命名违规：用户空间文件须符合语言习惯`
11. 若 `## 上下文` 章节存在，检查内容总长度（去格式化标记后纯文本字符数）是否 ≤ 1000。若超过 → 记录 `上下文章节超长（>1000 字符）`。
12. 若 `## 上下文` 章节存在，逐条检查内容中是否包含以下可被独立查找的内容原文：
    - 可通过文件路径定位的源码片段 → 记录 `上下文中包含源码原文，应替换为文件路径+行号引用`
    - 可通过文件名/目录名定位的路径信息 → 记录 `上下文中包含目录原文，应替换为路径引用`
    - 可通过 URL 定位的公开文档原文 → 记录 `上下文中包含文档原文，应替换为 URL+标题引用`
    - 可通过网络搜索获取的公开知识 → 记录 `上下文中包含可搜索知识原文，应替换为搜索关键词引用`
    - 只有当前上下文独有的分析产物（推理结论、综合判断、设计决策）可内联，不在此限制范围内
13. 提取全文所有 `[[link]]`，逐个校验：
    - link 路径仅限 `define/`、`method/`、`state/`。否则记录 `非法引用域：<link>`。
    - 通过 [[method/resolvebase/lookup]] 验证每个 link 目标存在。若不存在，记录 `死链：<link>`。
14. 若违规列表为空，返回"校验通过"。否则返回"校验失败"并附违规列表。
