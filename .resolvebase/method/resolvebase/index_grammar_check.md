# 校验索引文件语法

## 输入
- `目标`：待校验的 index 文件路径（[[define/resolvebase/index-l1|一级索引]]、[[define/resolvebase/index-l2|二级索引]]或子索引）

## 输出
- 校验通过
- 或：校验失败 + 违规项列表

## 过程

1. 读取 `目标` 文件，按行解析。
2. 判断类型：一级索引（`index/index.md`）、二级索引（`index/<类目名>.md`）或子索引（`index/<类目名>-<子领域>.md`）。

### 链接语法校验（所有类型通用）

3. 遵循 [[define/resolvebase/link-syntax|链接语法]]，提取文件中所有 `[[...]]` 模式，逐个检查：
   - 若出现 `[[...]](...)` 模式 → 记录 `链接语法错误：管道符应替换括号，正确格式 [[path|text]]`。
   - 若出现 `[...](...)` 模式（Markdown 链接） → 记录 `链接语法错误：不应使用 Markdown 链接，请用 [[wikilink]]`。

### 文件命名校验（所有类型通用）

4. 遵循 [[define/resolvebase/file-naming|文件命名规则]]：索引文件（`index/*.md`）属于 Resolvebase 系统文件，文件名须为小写英文连字符格式。否则 → 记录 `命名违规：系统索引文件须英文命名`

### 一级索引校验

5. 检查每个条目是否包含：
   - `# <类目名>`
   - `description: <说明>`
   - `link: [[index/<类目名>]]`
6. 验证每个 `link` 指向的二级索引文件存在。不存在 → 记录 `死链：<link>`。
7. 若有条目缺少 `description` 或 `link` → 记录 `条目不完整：<类目名>`。
8. **层级违规检查**（遵循 [[define/resolvebase/index-l1|一级索引]]约束）：
   - 若出现 `# Define`、`# Method`、`# State` 分组 → 记录 `层级违规：一级索引含类型分组`。
   - 若任何 link 指向非 `index/` 路径（如 `[[define/...]]`、`[[method/...]]`、`[[state/...]]`）→ 记录 `层级违规：一级索引直接引用 <link>`。

### 二级索引校验

9. 从文件名提取类目名 `<类目名>`（去掉 `index/` 前缀和 `.md` 后缀）。
10. 检查是否按类型分组：`# Define`、`# Method`、`# State`（可为空分组，但不能缺失分组标题）。
11. 每个条目检查：
    - `## <文件名>`
    - `description: <概括>`
    - `link: [[<类型>/<类目名>/<文件名>]]`
    - `require: [[...]], [[...]]`（可选）
12. 条目缺少 `description` 或 `link` → 记录 `条目不完整：<文件名>`。
13. 条目混杂（如 Define 分组下有 Method 类型的 link）→ 记录 `类型分组错位：<文件名>`。
14. **层级违规检查**（遵循 [[define/resolvebase/index-l2|二级索引]]约束）：
    - 检查每个条目的 `link`，若 `<类型>/<类目名>` 部分与当前索引所属类目不一致 → 记录 `层级违规：跨类目引用 <link>`。
    - 二级索引下的子索引条目（`link: [[index/...]]`）必须确保子索引内的条目同样属于本类目（若存在则递归校验）。
15. 提取所有 `[[link]]`（含 `link:` 和 `require:` 字段中的），逐个通过 [[method/resolvebase/lookup]] 验证目标存在。不存在 → 记录 `死链：<link>`。
16. 验证该二级索引从 `index/index.md` 出发可达。若不可达 → 记录 `索引孤立：无主入口可达路径`。

### 子索引校验

17. 同二级索引校验步骤 9-16，类目名从文件名 `<类目名>-<子领域>` 中提取前半部分作为所属类目。
18. 若违规列表为空，返回"校验通过"。否则返回"校验失败"并附违规列表。
