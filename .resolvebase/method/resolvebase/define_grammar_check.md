# 校验定义文件语法

## 输入
- `目标`：待校验的 [[define/resolvebase/define|定义]]文件路径

## 输出
- 校验通过
- 或：校验失败 + 违规项列表

## 过程

1. 读取 `目标` 文件，按行解析。
2. 检查首行是否以 `# ` 开头。否则记录 `缺少名称行`。
3. 检查是否无以下禁止章节（`## ` 开头）：
   - `## 依赖` → 记录 `禁止独立依赖章节，依赖须以 [[link]] 内联在正文中`
   - `## 输入`、`## 输出`、`## 过程` → 记录 `[[define/resolvebase/define|定义文件]]包含[[define/resolvebase/method|方法]]专属章节`
4. 链接语法校验（遵循 [[define/resolvebase/link-syntax|链接语法]]）：提取全文所有 `[[...]]` 模式，检查：
   - 若出现 `[[...]](...)` → 记录 `链接语法错误：应使用管道符 [[path|text]]`
   - 若出现 `[...](...)` (Markdown 链接) → 记录 `链接语法错误：应使用 [[wikilink]]`
5. 文件命名校验（遵循 [[define/resolvebase/file-naming|文件命名规则]]）：
   - 若 `目标` 属于 Resolvebase 系统文件路径（`define/resolvebase/`、`method/resolvebase/`、`state/resolvebase/`），文件名须为小写英文连字符格式。否则 → 记录 `命名违规：系统文件须英文命名`
   - 若 `目标` 不属于系统文件路径，文件名须符合用户语言习惯（中文交互用中文命名）。否则 → 记录 `命名违规：用户空间文件须符合语言习惯`
6. 提取全文所有 `[[link]]`，逐个校验：
   - 所有 link 必须以 `define/` 开头。否则记录 `非法引用域：<link>`。
   - 通过 [[method/resolvebase/lookup]] 验证每个 link 目标存在。若不存在，记录 `死链：<link>`。
7. 检查内容是否为断言式（"A 是 B"），非操作指引。若发现动词开头的指令句，记录 `疑似操作指引：<行内容>`。
8. 检查是否一个文件只定义一个概念。若标题含"和"/"与"/"及"等并列连词，记录 `可能混合多概念`。
9. 若违规列表为空，返回"校验通过"。否则返回"校验失败"并附违规列表。
