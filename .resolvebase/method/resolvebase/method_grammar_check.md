# 校验方法文件语法

## 输入
- `目标`：待校验的 [[define/resolvebase/method|方法]]文件路径

## 输出
- 校验通过
- 或：校验失败 + 违规项列表（每项一条）

## 过程

1. 读取 `目标` 文件，按行解析。
2. 检查首行是否以 `# ` 开头。否则记录 `缺少名称行（# <名称>）`。
3. 检查是否按顺序包含 `## 输入`、`## 输出`、`## 过程` 三个章节标题。缺少任一，记录 `缺少章节：<章节名>`。
4. 检查除名称行和上述三个章节外，是否无其他 `## ` 开头的行。若有，记录 `多余章节：<章节名>`。
5. 检查 `## 输入` 下的条目是否使用 `` `变量名`：<约束> `` 格式。不符合的条目记录。
6. 检查 `## 输出` 是否至少包含成功路径和以"或："开头的失败路径。
7. 检查 `## 过程` 下的步骤是否以 `数字. ` 开头（如 `1. `、`2. `）。不符合的记录。
8. 链接语法校验（遵循 [[define/resolvebase/link-syntax|链接语法]]）：提取全文所有 `[[...]]` 模式，检查：
   - 若出现 `[[...]](...)` → 记录 `链接语法错误：应使用管道符 [[path|text]]`
   - 若出现 `[...](...)` (Markdown 链接) → 记录 `链接语法错误：应使用 [[wikilink]]`
9. 文件命名校验（遵循 [[define/resolvebase/file-naming|文件命名规则]]）：
   - 若 `目标` 属于 Resolvebase 系统文件路径，文件名须为小写英文连字符格式。否则 → 记录 `命名违规：系统文件须英文命名`
   - 若 `目标` 不属于系统文件路径，文件名须符合用户语言习惯。否则 → 记录 `命名违规：用户空间文件须符合语言习惯`
10. 提取 `## 过程` 中所有 `[[link]]`，逐个校验：
    - link 路径必须以 `define/` 或 `method/` 开头。否则记录 `非法引用域：<link>`。
    - 通过 [[method/resolvebase/lookup]] 验证每个 link 目标存在。若不存在，记录 `死链：<link>`。
11. 若违规列表为空，返回"校验通过"。否则返回"校验失败"并附违规列表。
