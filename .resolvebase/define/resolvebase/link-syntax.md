# 链接语法

## 本质

链接语法是 Resolvebase 空间中引用其他文件的统一格式。它基于 Obsidian 的 `[[wikilink]]` 语法，确保引用语义精确且对人类可读。

## 基本格式

```
[[文件路径]]
```

仅链接到文件，文件名直接作为显示文本。

## 带显示文本

```
[[文件路径|显示文本]]
```

链接到文件，但以**显示文本**作为人类可读的呈现内容。显示文本用于本地化或语义修饰，使阅读者在当前语境中直接理解含义，而无需跳转。

**管道符 `|`** 是关键分隔符：左侧是路径，右侧是显示文本。

**错误示例**：
- `[[define/resolvebase/index]](索引)` —— `]]` 不是合法语法，管道符连接才是正确方式。

**正确示例**：
- `[[define/resolvebase/index|索引]]` —— 链接到索引定义文件，显示为"索引"
- `[[define/resolvebase/index-l1|一级索引]]` —— 链接到一级索引定义，显示为"一级索引"

## 带锚点

```
[[文件路径#标题|显示文本]]
```

链接到文件内的特定标题/段落，进一步精确定位语义。

**示例**：
- `[[define/resolvebase/index#本质|索引]]` —— 链接到索引定义文件的"本质"段落
- `[[method/resolvebase/manage-index#添加类目|添加类目]]` —— 链接到管理索引方法文件的"添加类目"章节

其中 `#标题` 部分是可选的。

## 在 Resolvebase 中的应用规则

- [[define/resolvebase/define|定义文件]]中引用其他定义时，使用 `[[define/...|显示文本]]`
- [[define/resolvebase/method|方法文件]]中引用定义或方法时，使用 `[[define/...|显示文本]]` 或 `[[method/...|显示文本]]`
- [[define/resolvebase/state|状态文件]]中引用时，使用 `[[define/...|显示文本]]`、`[[method/...|显示文本]]` 或 `[[state/...|显示文本]]`
- [[define/resolvebase/index|索引文件]]中条目 link 为纯路径格式 `[[类型/类目/文件名]]`（无管道符）

## 禁止模式

- `[[...]](...)` —— 括号外挂显示文本，非法语法
- `[显示文本](路径)` —— Markdown 标准链接语法，在 Resolvebase 中不使用，避免与 Obsidian wikilink 混淆
