# 定义文件

## 本质

定义文件是 Resolvebase 空间中**最原子的知识单元**，对应代码中的常量（`const`）、类型（`type`/`interface`）或枚举（`enum`）。

## 核心特征

- **不可变性**：定义文件的内容是不可变的陈述。一旦确立，其核心语义不应被覆盖。若语义发生根本变化，应创建新定义文件而非原地修改。
- **被多次引用**：定义文件之所以存在，是因为其承载的知识会被 ≥ 2 个其他文件引用（或明确预期会被引用）。如果仅被引用一次，应内联到引用方中。
- **非公开可查**：定义文件不能储存常见、公开、可查的信息（如某 API 官方文档、维基百科条目）。它必须是特定于本项目或本协作上下文的约定/共识/经验。

## 内容边界

定义文件的内容必须是**断言式**的——"A 是 B"、"X 遵循 Y 规则"——不能是操作指引（那属于 [[define/resolvebase/method|方法文件]]），也不能是运行时快照（那属于 [[define/resolvebase/state|状态文件]]）。每句话应可被验证真伪。

## 引用约束

定义文件自身**可引用其他定义文件**（`[[define/...]]`），但**禁止引用**[[define/resolvebase/method|方法文件]]、[[define/resolvebase/state|状态文件]]和 [[define/resolvebase/index|索引文件]]。
